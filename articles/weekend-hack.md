这个周末做了一件一直想做但迟迟没动手的事：搭建个人博客。

说来惭愧，作为一个开发者，自己的个人站点却一直处于"在建中"的状态。每次想动手，都会陷入各种选择困难：用什么框架？什么主题？要不要上 CMS？要不要支持评论？

这次我决定做一个"最小可行产品"（MVP）。

## 技术选型

最终的选择非常简单：

- 纯 HTML + CSS + JavaScript
- 不使用任何框架或构建工具
- 不使用数据库或后台
- 所有页面都是静态的 .html 文件

做出这个选择之后，整个搭建过程变得异常顺畅。没有环境配置的烦恼，没有依赖版本冲突，没有构建失败需要排查——就是写 HTML 和 CSS 而已。

## 设计思路

设计上遵循几个原则：

- **内容优先**——排版以阅读体验为中心，字体、行距、留白都为阅读服务
- **简洁克制**——不使用多余的装饰元素，每个设计决定都有目的
- **响应式**——在手机和电脑上都有良好的阅读体验

我选择了一种暖色调的橙色作为 accent 色，搭配大面积留白和细腻的边框，营造出温暖而干净的视觉感受。

## 一些感受

在这次搭建过程中，最大的体会是：**限制反而带来自由**。

当决定使用最朴素的技术栈之后，我不再需要做各种权衡和选择，只需要专注于真正重要的事情——内容和设计。也许这就是"少即是多"的最好体现。

> 完美不在于无以复加，而在于无可删减。——安托万·德·圣埃克苏佩里
